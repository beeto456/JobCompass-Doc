Part 1: Positioning Statement

"For active job seekers who are applying to multiple roles and tailoring their resumes for different job descriptions..."

JobCompass is a job application tracking and resume management app that helps users clearly track their application pipeline, monitor progress, and organise job-specific resumes in one place.

"Unlike manual spreadsheets, notes apps, or scattered folders, JobCompass combines application status tracking, visual pipeline analytics, and resume-job linking into a single focused workflow designed specifically for job search management."

Core Differentiators

  • Active Document Vault Integrations
  • Automatic Calendar Milestone Syncing
  • Context-Match Index Analytics

Part 4: Demand / Value Hypothesis

Problem Scenario

Job seekers must reliably track multiple job applications simultaneously—including job titles, organizations, key timelines, descriptions, and current stages—so they can manage their search with confidence and keep opportunities moving forward.

Current Alternatives

Generic data spreadsheets, local text note files, browser bookmark lists, clogged email search queues, or simple unassisted memory. These methods are flexible but disjointed, making it difficult to gauge pipeline health or see which resume variation went where.

Demand/Value Hypothesis

"If we supply job seekers with a dedicated tracking ecosystem combining pipeline data visualizations, structural status tracking, job description storage, and contextual document linking, they will maintain logs more consistently and feel much more in control."

Part 7: Document Frequently Asked Questions

JobCompass solves the problem of fragmented job search management. Active job seekers often apply to multiple roles at once and rely on scattered, manual tools like spreadsheets, generic notes, browser bookmarks, and email folders. This quickly becomes unmanageable because job searches involve multiple moving parts: job descriptions, application dates, tailored resumes, follow-up timelines, and interview stages. JobCompass brings these scattered elements into one focused, cohesive tracking workspace.
It introduces a unified system featuring an interactive pipeline analytics dashboard, a status explorer, an interaction calendar, and a resume vault. The dashboard provides an instant, high-level overview of search progress, while the status explorer tracks individual opportunities as they shift across stages. The calendar handles critical interview and follow-up milestones, and the resume vault links specific CV versions directly to corresponding applications.
The target user is an active job seeker balancing multiple applications and managing tailored resume variations for different roles. It is built specifically for anyone who needs an organized way to track exactly what they submitted, where each application stands, and when upcoming interviews or follow-ups require attention.
While spreadsheets offer flexibility, they lack built-in pipeline visualizations, direct resume-to-job linking, and automated milestone tracking. Similarly, generic calendar apps store dates but disconnect them from the broader application context—such as the job description, the specific resume version used, current application status, and historical notes. JobCompass unites these needs into one tailored workflow.
The primary value proposition is bringing clear structure, confidence, and consistency to the job search. Instead of losing track of details across different applications, users have immediate, centralized visibility into their status records, submission histories, file versions, and upcoming action items.
If active job seekers are given a dedicated tool that combines application tracking, pipeline visualization, calendar-based milestone tracking, job description storage, and resume linking, then they will log applications more consistently, update statuses more frequently, follow up more reliably, and feel more in control of their job search process.
Success means users move beyond just logging an application to actively updating and maintaining their pipeline over time. This includes adjusting statuses, tracking calendar milestones, creating follow-up actions, and referencing linked resumes when preparing for interviews. A successful product becomes an essential part of the user's daily job search routine.
We track both behavioral and qualitative metrics. Behavioral metrics include form completion rates, status update frequencies, dashboard return patterns, calendar event creation, resume uploads, and feature interaction metrics. Qualitatively, we measure changes in user confidence, perceived organization, and interview readiness through feedback surveys.
The single most critical metric is successful job application management behaviour over time. While signups demonstrate initial interest, real value is proven when users repeatedly return across multiple sessions to update statuses, review their pipeline analytics, and organize upcoming interview milestones.
The core concept emerged from qualitative interviews, user surveys, and extensive desk research confirming a clear user preference for a unified tracking tool. To validate further, we run usability tests with our interactive prototype, evaluating how efficiently users complete key tasks like adding applications, scheduling interviews, uploading resumes, and linking files.
We assume job seekers find fragmented tracking painful enough to switch to a dedicated tool, and that they see clear value in linking resumes and tracking milestones within a single app. We also assume a strong preference for mobile access, as data shows many job seekers manage their search on the go. Testing helps ensure this added structure feels supportive rather than burdensome.
The primary risk is product abandonment if the platform begins to feel like tedious admin work for already exhausted job seekers. To mitigate this, the core logging workflow must remain incredibly lightweight. Optional details are tucked away using progressive disclosure, keeping fields like title and company simple and fast, while generating calendar milestones automatically.
Instead of launching users onto an empty dashboard, onboarding guides them directly to a single high-value action: logging their first job application. Once created, the app instantly demonstrates how that single log populates the pipeline overview and sets up calendar milestones, clearly showing the platform's value right from the start.
Engagement is measured by how consistently users update and maintain their application pipelines. We monitor the frequency of status changes, calendar views, resume links, and search queries. For JobCompass, high engagement isn't about long session durations, but rather quick, frequent, and productive check-ins.
By combining product analytics with direct user sentiment tracking. Success is proven when data shows steady pipeline updates and strong user retention, paired with qualitative feedback confirming that users feel more organized, less anxious, and more in control of their job search.
We carefully balanced completeness against simplicity. While an application can track dozens of unique data fields, forcing users to fill them all out upfront causes friction. We also balanced flexibility against structure—ensuring the app provides enough guidance to prevent the messiness of a spreadsheet, while remaining adaptable to individual workflows.
The MVP focuses on the core user loop: logging an application, viewing it within a visual pipeline, updating its status, and returning to manage details. The calendar component is kept simple, tracking only essential dates (applications, interviews, follow-ups) to avoid feature creep. The current live interactive prototype serves as this foundational MVP.
We look closely at the initial onboarding experience. If users drop off immediately, it usually means the entry form was too long, empty states felt confusing, or the immediate value wasn't clear. We address this by simplifying entry fields, sharpening our onboarding call-to-actions, and conducting user interviews to pinpoint friction points.
The calendar translates static application entries into timely, actionable schedules. Showing exactly when an application was sent, when an interview is coming up, or when a follow-up is due removes the mental burden of tracking dates across multiple platforms or relying entirely on memory.
We must avoid over-engineering the calendar into a general-purpose lifestyle planner, which dilutes our product focus. It must remain strictly dedicated to job search milestones. Testing also helps determine how much automation users prefer versus manual calendar customization.
It emphasized that UX design concepts must connect directly to measurable user behaviors. True organization isn't just a feeling; it's reflected in concrete actions like updating pipelines, linking resumes, and tracking tasks. The CX metrics framework helps bridge the gap between design choices and user actions.
I would run deeper validation rounds with active job seekers through focused usability tests. From there, I'd refine our tracking analytics to ensure every monitored event provides actionable product insights. Long-term features like notification systems, direct Google Calendar syncing, and automated form-filling would be prioritized based on continuous user feedback.
Google authentication provides a secure way to save and persist sensitive job search data in our cloud database, making it available across all user devices. While Guest Mode offers a safe, isolated sandbox for testing, data is saved locally to the browser cache and risks being wiped. Clear privacy messaging ensures users always know how their data is handled, stored, and managed.
